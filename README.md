## 設計の核(構造不変条件)

ご要望「全部無料・弁護士へつながない・申請フォーマットも・警察側書面もそのまま使える」を、
**抜け道のない構造**に落としている。三層(schema `:db/allowed` + lexicon `:const` + Python
`ValueError`)で表現不能にした不変条件:

- **G1 全て無料(cash≡0)** — 手数料・課金・購読は *表現できない*。`:support/cost-jpy` は
  `[0]` のみ許可。`analyze.py` は全ケースの被害者負担が **¥0** であることを assert する。
- **G2 本人作成・本人提出** — `:support/role` は `案内 / 起草補助 / 本人提出` のみ。
  `代理作成 / 代理提出` は表現できない。助は書面を**作る**が、署名・提出は**本人**。
  (行政書士法・弁護士法の独占業務を踏まない)
- **G3 警察authored不可** — 生成書面の作成者は `:member` のみ。**警察官名義の公文書は表現できない**
  (公文書偽造を構造的に排除)。「そのまま使える」= 警察官が*作業の土台にできる*完成度、の意。
- **G5 弁護士へつながない** — 有料 counsel への紹介は表現できない(`paidReferral [false]`)。
  無料の公的窓口のみ。

その他: G4 非裁定(詐欺認定はしない)/ G6 証拠は暗号化参照 / G7 no-server-key(本人署名)/
G8 Murakumo-only / G9 R0 は draft のみ(実提出は Council+operator gated)/ G10 窓口は要一次情報確認。

## 使う（誰でも）

**ブラウザだけで(インストール不要・端末内で完結)**: `app/index.html` を開く → 質問に答える →
書類一式が生成され、各書類を `.txt` 保存 / 印刷・PDF 保存できる。**通信は一切せず**、入力した情報は
端末の外に出ない(no-server-key / PII 非送信)。`did:web:etzhayyim.com:actor:tasuke` の actor 面。

被害に遭った本人が、ワンコマンドで **そのまま印刷して警察・銀行・プラットフォームに出せる書類一式**
を作れる(CLI):

```bash
cd methods
python3 intake.py                        # 質問に答えるだけ(EDN不要・誰でも使える)
python3 packet.py --case c-fund-1        # seed の被害ケースで生成
python3 packet.py --file my-case.edn     # 自分の被害内容(EDN)で生成
# → out/packet-<id>/ に 00-COVER.md(初動チェックリスト+無料窓口+期限) と
#   各書類(被害届/被害状況報告書/証拠目録/被害額算定書/銀行組戻し依頼/復旧手順)が出力される
```

生成物はすべて **本人が署名して提出** する書類で、費用は **¥0**。

## 正直な R0 の限界

設計 + オフライン生成のみ。実際の届出・送付・口座凍結依頼の送信・アカウント操作は **行わない**
(すべて G9 で Council Lv6+ + operator gated)。窓口・根拠法令・法定期限は `:representative` の
seed であり、**一次情報での確認が必要**。詐訟代理が必要な事案は所掌外(有料 counsel には繋がない —
無料の公的相談窓口を案内するに留める)。
